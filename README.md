# Александр Рост — сайт поэта

Минималистичный сайт на чистом HTML. Стихи хранятся в `poems.js`
и обновляются автоматически через GitHub Actions + Playwright.
Никаких платных API не нужно.

---

## Структура файлов

```
├── index.html                        # Сайт
├── poems.js                          # База стихотворений (генерируется автоматически)
├── scripts/
│   ├── fetch_poems.py                # Парсер stihi.ru через Playwright
│   └── poems_cache.json             # Кэш — уже скачанные стихи
└── .github/workflows/
    └── update-poems.yml              # GitHub Actions: расписание + деплой
```

---

## Первый запуск

### 1. Создать репозиторий на GitHub

```bash
git init
git add .
git commit -m "init"
git remote add origin https://github.com/ВАШ_НИК/РЕПО.git
git push -u origin main
```

### 2. Включить GitHub Pages

GitHub → **Settings** → **Pages** → Source: **GitHub Actions**

### 3. Запустить вручную

GitHub → **Actions** → **Update Poems & Deploy** → **Run workflow**

Первый запуск скачает все стихи (~10 мин).
Дальше Action работает сам — каждый день в 09:00 МСК.

> Никакие секреты и API-ключи не нужны.

## Как добавить новое стихотворение

Есть два способа.

---

### Способ А — автоматически (рекомендуется)

Когда Александр публикует новое стихотворение на stihi.ru,
Action подхватит его сам на следующий день.

Если нужно сразу — запустите Action вручную:

```
GitHub → Actions → Update Poems & Deploy → Run workflow
```

---

### Способ Б — вручную через GitHub

Если стихотворение не на stihi.ru, добавьте его напрямую в `poems.js`.

**Шаг 1.** Откройте файл `poems.js` на GitHub (нажмите карандаш ✏️)

**Шаг 2.** Найдите строку:
```js
const POEMS = [
```

**Шаг 3.** Сразу после неё добавьте новое стихотворение:
```js
const POEMS = [
  { date: "22.05.2026", title: "Название стихотворения", text: "Первая строка\nВторая строка\nТретья строка" },
  // ... остальные стихи
```

**Правила форматирования:**

| Поле    | Формат | Пример |
|---------|--------|--------|
| `date`  | `ДД.ММ.ГГГГ` | `"22.05.2026"` |
| `title` | Строка | `"Коллаборация"` |
| `text`  | Строки через `\n` | `"Строка 1\nСтрока 2\nСтрока 3"` |

Строфы разделяются двойным переносом `\n\n`:
```js
text: "Первая строфа\nпервая строфа продолжение\n\nВторая строфа\nвторая строфа продолжение"
```

**Шаг 4.** Нажмите **Commit changes** → сайт обновится автоматически.

---

### Способ В — вручную локально

```bash
# Откройте poems.js в любом редакторе (VS Code, Sublime и т.д.)
# Добавьте стихотворение в начало массива POEMS
# Сохраните файл

git add poems.js
git commit -m "добавлено стихотворение «Название»"
git push
```

## Расписание обновлений

По умолчанию Action запускается **каждый день в 09:00 МСК**.

Изменить расписание можно в `.github/workflows/update-poems.yml`:
```yaml
- cron: '0 6 * * *'    # каждый день в 06:00 UTC (09:00 МСК)
- cron: '0 6 * * 1'    # только по понедельникам
- cron: '0 6 1 * *'    # первого числа каждого месяца
```

## Локальный запуск скрипта

```bash
pip install playwright
playwright install chromium
python scripts/fetch_poems.py
```

Скрипт обновит `poems.js` и `scripts/poems_cache.json`.
Закоммитьте оба файла.
